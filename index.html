CSS selectors and specificity

What is specificity?

Specificity is the algorithm used by browsers to determine the CSS declaration that is the most relevant to an element, which in turn, determines the property value to apply to the element. The specificity algorithm calculates the weight of a CSS selector to determine which rule from competing CSS declarations gets applied to an element.

Possible selectors in order of specifity

Selector name Syntax Description
Universal selector * It matches all elements in the HTML file. Since this selector matches all elements, it is the least specific one and will be overshadowed by other selectors.
TYPE selector HTML node name It matches elements by node name. In other words, it selects all elements of the given type within a document such as p, h1, and td. This selector is less specific than the CLASS selector and ID selector and will be overshadowed by them.
CLASS selector .selector It matches elements based on the contents of their class attribute. This selector is usually used to match multiple elements that have the same characteristics. It is less specific than the ID selector, but more specific than the TYPE selector.
ID selector #selector It matches an element based on the value of the element's id attribute. This selector is used to match one and only one HTML element, because of that is the more specific of them all and will overshadow any other selector.
